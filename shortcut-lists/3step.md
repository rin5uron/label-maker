# 🚀 Next.js開発 課題解決3選（実践編）

## トラブル1：Port 3000 is already in use（ポート占有エラー）

### 症状：
```bash
Error: Port 3000 is already in use
```

### 原因：
- 前回のNext.jsプロセスが残っている
- 別のアプリが3000番ポートを使用中

### 解決方法：

#### ステップ1：ポートを使っているプロセスを特定
```bash
lsof -i :3000
```

出力例：
```
COMMAND   PID   USER
node    12345  yourname
```

#### ステップ2：プロセスを強制終了
```bash
kill -9 12345
```

#### ワンライナー（一発解決）：
```bash
kill -9 $(lsof -t -i:3000)
```

---

## トラブル2：ビルドが終わらない・フリーズした

### 症状：
`npm run build` や `npm run dev` が固まって動かない

### 解決方法：

#### ステップ1：実行中のNodeプロセスを確認
```bash
ps aux | grep node
```

出力例：
```
yourname  12345  node /path/to/next
yourname  67890  node /path/to/next
```

#### ステップ2：該当プロセスを終了
```bash
kill -9 12345 67890
```

#### または全部まとめて終了：
```bash
pkill -9 node
```
⚠️ 注意：他のNode.jsアプリも止まります

---

## トラブル3：前に実行したコマンドが思い出せない

### 症状：
- さっき実行した長いコマンド、もう一回使いたい
- 「あれ、どうやって起動したっけ？」

### 解決方法：

#### 方法1：履歴から検索
```bash
history | grep "npm"
```

#### 方法2：Ctrl + R で逆検索（最速！）
1. `Ctrl + R` を押す
2. `npm` と入力
3. 該当コマンドが表示される
4. `Enter` で実行 / `Ctrl + R` 連打で履歴を遡る

#### 方法3：矢印キー ↑ で履歴を辿る
- `↑` を押すたびに1つ前のコマンドが表示される

## 💡 予防策：開発終了時のお作法

### 開発終了時は必ず Ctrl + C でプロセス終了！
```bash
# 開発サーバー起動中
npm run dev

# 終了時：Ctrl + C を押す
^C
```

### ターミナルを閉じる前に確認：
```bash
# 実行中のプロセスがないか確認
ps aux | grep node

# 何も表示されなければOK！
```
